Configure the Resource path

Metalab needs quite a few external modules/software and self-made databases to run, for protein identification and taxon/function mapping. Make sure you set up the path before running. We deliberately isolate the resource folder in order to make the update/upgrade of Metalab easier. By doing so you only need to download core files (usually only several Megabytes ) from the newly released cored and replace the old one, or you can keep all the versions in place utilizing the same resources, without the need of multiple copies. It recommended that you organize all resources in the You can put the resource folder anywhere, preferably outside the Metalab software folder.

After downloading all files, the first thing you need to do is to set up the resources. Refer to the following figure for more information.

Place MaxQuant in mq_bin

Metalab employs MaxQuant for closed search workflows. See the relationship with MaxQuant. We do not include a complete copy in the software distribution because of the copyright issue. Sorry for the inconvenience.

If you happen to have a relatively new copy of MaxQuant, congrats! Otherwise, users need to download Maxquant by themselves following the link https://maxquant.org/.We acknowledge the hard and wonderful work of Maxquant team. follow the link to read more about Maxquant and download

Maxquant is fast updating. We cannot make sure each version is fully compatible. The version we mostly tested is 1.6.3.4. This version also runs smoothly in mono on Linux. If you have some issue with the MaxQuant run (close search step), you can try this version.

Generate a Maquant parameter template for closed search

Different versions of MaxQuant have a different format of config parameter (file mqpar.xml ) for the command line run for MetaLab to evoke. We choose an easy way to work around, which is that the user has to put a mqpar.xml into the \Resources\mq_bin directory. MetaLab will use that parameter as a template to generate new parameters.

Users can copy any already automatically generated mqpar.xml from the raw files working folder if they have run MaxQuant search before. Or they can simply generate a new parameter following the steps by run the Resources\mq_bin\MaxQuantGui.exe

  • Be sure to see this mqpar.xml in \Resources\mq_bin directory before any closed search, otherwise, MetaLab will report an error :
    • Closed search: parameter file “mqpar.xml” was not found in Resources\mq_bin\mqpar.xml, please generate it by MaxQuant GUI
  • This mqpar.xml is  MaxQuant version-specific
  • If you want to switch MaxQuant version, re-generate this file

Taxon mapping file

Metalab can do taxon mapping right after database search (peptide and protein identification). We provide two options for this function.

For the first option, users need to download pep2taxon.gz to put it in the pept2taxa folder. This file can be either compressed or unzipped, which helps to speed up a bit. You can also choose to use unipept to do the taxon analysis if you have internet access.

Configure the Memory

The default memory/RAM usage is 4GB .For large scale data, edit the MetaLab.bat (right click>>edit)to change the maximum RAM usage to a larger value, like here 12g

java -Xmx12g -jar MetaLab.jar -gui
