Meta info set up

Here in Metalab, meta-information mean grouping information.

For example, we have a small project with two groups, 3 replicates each. We would like to set up 6 experiments, which will give us information about reproducibility. By doing so, you will be able to get 6 columns of quantification information. This is the Experiment information. We also need another grouping information, to tell the first 3 raw files/samples are from the control group, while the rest 3 files/samples are from the treatment group, to do the statistic analysis, e.g. looking for differentially expressed features among the two groups.

Meta info is not directly written/integrated into the result file, instead, it will be written into a separate .txt file, which will facilitate the iMetaReport.

The meta is optional, it does not affect anything during the automatic data analysis(peptide and protein id, as well as taxon mapping). You can add up to 5 groups of meta.

Only the first groups of meta will be used for MetaReport (If there is meta info). If you do not have meta information, leve the number of type as 0, otherwise, reports might not be generated correctly with void meta/grouping information.
